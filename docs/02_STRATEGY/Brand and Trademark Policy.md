# Brand and Trademark Policy

> ⚠️ Это **draft governance direction**, а не финальная юридическая консультация. Перед применением необходима консультация с юристом, специализирующимся на интеллектуальной собственности.

---

## Purpose

Этот документ определяет политику использования бренда **Rashamon** — имени, логотипов и официальной идентичности.

**Ключевой принцип**: код открыт, бренд управляется.

---

## Distinction: Code License vs Brand Identity

### Code License

- Исходный код Rashamon лицензирован под открытой лицензией (TBD: MIT/Apache-2.0/GPL)
- Любой может форкнуть, модифицировать, использовать код
- Лицензия даёт права на **код**

### Brand Identity

- Имя «Rashamon», логотипы, официальный сайт, каналы — **управляются отдельно**
- Форк кода **не даёт** права использовать имя «Rashamon» для распространения форка
- Это стандартная практика: Mozilla Firefox, Blender, Red Hat, Canonical Ubuntu

## Fork Policy

### Код

- ✅ Форк кода разрешён лицензией
- ✅ Модификация разрешена
- ✅ Распространение модификаций разрешена (по условиям лицензии)

### Бренд

- ❌ Форк **не может** использовать имя «Rashamon» как своё основное название
- ✅ Форк **может** сказать «based on Rashamon» или «fork of Rashamon»
- ✅ Форк **должен** использовать собственное имя и брендинг
- ❌ Форк **не может** использовать официальные логотипы без модификации

### Примеры

| Сценарий | OK? |
|---|---|
| «MyDraw — fork of Rashamon Draw» | ✅ |
| «Rashamon Community Edition» | ⚠️ Требуется разрешение |
| «Rashamon Pro» (сторонний продукт) | ❌ |
| «NewDraw (based on Rashamon Draw)» | ✅ |

## Official Builds

Только официальные билды от мейнтейнеров Rashamon могут:

- Использовать имя «Rashamon» без квалификаторов
- Использовать официальные логотипы
- Распространяться через официальные каналы
- Именоваться «Rashamon Draw» и т.д.

Сторонние билды должны:

- Использовать другое имя
- Указать «based on Rashamon»
- Не использовать официальные логотипы

## Governance Body

На текущий момент governance — **responsibility of the original maintainer** (kamelot-exe).

По мере роста проекта:

1. Создать **Brand Committee** (2-3 человека)
2. Определить процесс принятия решений по бренду
3. Опубликовать Brand Guidelines публично

## Enforcement

### Подход

1. **Notice** — первое уведомление нарушителю
2. **Discussion** — диалог о решении
3. **Formal request** — официальное требование прекратить
4. **Legal action** — крайняя мера (если trademark зарегистрирован)

### Принципы

- Дружелюбие прежде: большинство нарушений — непреднамеренные
- Пропорциональность: маленькие проекты не target
- Community-first: репутация проекта важнее агрессии
