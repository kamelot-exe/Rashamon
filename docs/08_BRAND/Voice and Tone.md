# Voice and Tone

Голос и тон коммуникаций Rashamon.

---

## Voice (постоянный)

Голос Rashamon — **экспертный, прямой, уважительный**.

| Аспект | Описание |
|---|---|
| Экспертный | Мы знаем что делаем; не извиняемся |
| Прямой | По существу; без fluff; конкретика |
| Уважительный | К пользователю, контрибьютору, времени |
| Спокойный | Без hype, без panic |
| Честный | Признаём ограничения; не обещаем невозможного |

---

## Tone (вариативный)

Тон адаптируется к контексту:

| Контекст | Тон | Пример |
|---|---|---|
| Документация | Нейтральный, инструктивный | «Нажмите Ctrl+S для сохранения» |
| Error message | Прямой, helpful | «Не удалось сохранить файл. Проверьте права доступа» |
| Release notes | Фактологический | «Добавлена поддержка SVG export» |
| GitHub issue | Технический, конкретный | «Crash при undo после delete — confirmed, working on fix» |
| Blog post | Экспертный, narrative | «Почему мы выбрали Tauri 2» |
| Community chat | Дружелюбный, но профессиональный | «Спасибо за PR! Есть один комментарий...» |
| Brand announcement | Уверенный, вдохновляющий | «Rashamon Draw alpha — первый шаг» |

## Writing Guidelines

### Do

| Правило | Пример |
|---|---|
| Пишите конкретно | «Добавлена поддержка SVG export в PNG» |
| Используйте активный залог | «Система сохраняет документ» |
| Будьте кратки | Одно предложение — одна мысль |
| Определите термины | «Scene Graph — иерархия объектов (см. [[Glossary]])» |
| Пишите на русском (основной) | С техническими терминами на английском |

### Don't

| Правило | Пример |
|---|---|
| Не используйте hype | «Революционный инструмент!» |
| Не извиняйтесь чрезмерно | «К сожалению, к сожалению, мы, к сожалению...» |
| Не будьте размытыми | «Возможно, когда-нибудь, потенциально...» |
| Не используйте jargon без объяснения | «Мы used the DAG approach» (что такое DAG?) |
| Не пишите marketing fluff | «Best-in-class, cutting-edge, next-gen» |

## Error Messages

### Format

```
[Что случилось] [Почему] [Что делать]
```

### Примеры

| ❌ Плохо | ✅ Хорошо |
|---|---|
| «Ошибка!» | «Не удалось открыть файл: формат не поддерживается. Поддерживаемые форматы: .rdraw, .json» |
| «Произошла ошибка при сохранении» | «Не удалось сохранить документ: нет прав на запись в указанную директорию» |
| «Something went wrong» | «Рендеринг завершился с ошибкой: недостаточно видеопамяти. Попробуйте уменьшить область просмотра» |

## UI Text

| Элемент | Стиль |
|---|---|
| Кнопки | Глагол: «Сохранить», «Экспорт» |
| Меню | Существительное: «Файл», «Правка» |
| Tooltips | Краткое действие: «Выделить объекты (V)» |
| Notifications | Результат: «Документ сохранён» |
| Errors | Проблема + решение |

## Documentation Text

| Раздел | Стиль |
|---|---|

| Architecture | Technical, precise, diagrams |
| User Guide | Step-by-step, screenshots |
