# Product Philosophy

Философия Rashamon — это набор **принципов-фильтров**, через которые принимается каждое продуктово-техническое решение.

---

## Core Principles

### 1. Серьёзный инструмент, а не игрушка

Rashamon создаётся для **профессионалов и продвинутых любителей**, которым нужен реальный рабочий инструмент.

- Никаких упрощений, которые ломают профессиональные сценарии
- Глубокие функции важнее красивых онбордингов
- Keyboard-first UX: всё можно сделать с клавиатуры

### 2. Неразрушающая философия (Non-destructive by default)

Каждое действие в Rashamon должно быть **обратимым, переиспользуемым и параметрическим**.

- Операции — это узлы в графе, а не деструктивные изменения
- История — ветвящийся граф, а не линейный undo/redo
- Эффекты и трансформации — параметрические, а не baked-in

### 3. Семантика поверх пикселей

Холст Rashamon — это **структурированные данные**, а не плоское изображение.

- Объекты несут метаданные: тип, роль, связи, теги
- Семантический слой позволяет автоматизацию, поиск, генерацию
- Визуальное представление — проекция данных, а не сами данные

### 4. Модульность на всех уровнях

- Приложение = ядро + модули
- Плагины — первоклассные граждане с доступ к API ядра
- Каждый продукт Rashamon — отдельный модуль над общей платформой

### 5. Offline-first, быстро, локально

- Работает без интернета
- Не требует аккаунтов, облаков, подписок
- Все данные — на машине пользователя

### 6. Открытость и форкабельность

- Код открыт, лицензия разрешает форки
- Бренд и официальная идентичность — под управлением сообщества
- Форк кода ≠ право использовать имя «Rashamon» официально

### 7. Linux-native мышление

- Не «port с Windows», а «сделано для Linux»
- Wayland, системные темы, pkgmgr-пакетизация
- Уважение к Linux-философии: pipes, config files, CLI

### 8. AI — ассистент, а не замена

AI используется **только там, где даёт реальное преимущество**:

- Семантическая тегировка ассетов
- Предложения по оптимизации
- Автоматизация рутины

AI **не** заменяет художника и **не** генерирует контент по умолчанию.

## Anti-Principles (чего мы НЕ делаем)

| Не делаем | Почему |
|---|---|
| Не клонируем Adobe UI | Наша цель — лучшие workflow, а не копирование чужих решений |
| Не делаем подписку | Подписка создаёт стимул ломать, а не чинить |
| Не делаем «всё для всех» | Фокус на профессионалах, а не на массовой аудитории |
| Не жертвуем скоростью ради фич | Быстрый MVP лучше медленного «полного» продукта |
| Не блокируем форки | Открытый код без форкабельности — это маркетинг, а не open source |

---

## Key Decisions

| Принцип | Влияние на продукт |
|---|---|
| Non-destructive | Графовая история, параметрические эффекты |
| Semantic canvas | Структурированная модель данных, не flat raster |
| Offline-first | Нет cloud-sync в MVP, локальное хранение |
| Keyboard-first | Командная палитра, хоткеи на всё |
