# Differentiators

Уникальные особенности Rashamon, которые отличают его от существующих решений.

---

## 1. Семантический канвас

### Что это

Холст Rashamon — это **структурированное графовое представление объектов**, а не плоский набор пикселей или векторов.

Каждый объект на канвасе несёт:
- **Тип**: shape, text, image, group, symbol, guide
- **Метаданные**: имя, теги, описание, custom properties
- **Связи**: parent-child, references, constraints
- **Семантическую роль**: background, foreground, UI-element, annotation

### Почему это важно

| Без семантики | С семантикой |
|---|---|
| «Нарисован прямоугольник» | «Это кнопка интерфейса» |
| Машина не понимает контекст | Автоматизация понимает, что делать |
| Ручная организация | Автоматическая категоризация |

### Примеры использования

- «Выделить все UI-элементы» → семантический запрос
- «Заменить все кнопки на новую версию» → batch-операция
- «Экспортировать только слой макета» → фильтрация по роли

## 2. Графовая история (Branching Snapshots)

### Что это

История проекта — это **направленный ациклический граф (DAG)**, а не линейная последовательность.

- Каждый шаг — узел в графе
- Можно создавать **ветки** от любой точки истории
- Можно **сравнить** любые два узла
- Можно **слить** ветки или выбрать одну из альтернатив

### Почему это важно

| Linear undo/redo | Branching history |
|---|---|
| Отменил → потерял контекст | Ветка → можно вернуться |
| Один путь назад | Множество альтернатив |
| «А что если...» = риск | «А что если...» = ветка |

### Примеры использования

- «Попробовать другой цвет фона» → ветка, не теряя оригинал
- «Сравнить три варианта лэйаута» → три ветки рядом
- «Вернуться к состоянию час назад, но сохранить текущее» → snapshot + continue

## 3. Неразрушающее редактирование by default

### Что это

Каждая операция — **параметрический узел**, а не деструктивное изменение.

- Трансформации хранятся как параметры, не как baked-in результат
- Эффекты — цепочка узлов, каждый со своими настройками
- Можно вернуться и изменить любой узел в цепочке

### Сравнение

| Photoshop | Rashamon |
|---|---|
| Flatten → данные потеряны | Flatten → projection, данные целы |
| Smart Objects — опция | Non-destructive — default |
| Adjustment Layers — частично | Всё — параметрические узлы |

## 4. Автоматизация и hooks

### Что это

Rashamon предоставляет **хуки и макросы** для автоматизации рутинных операций.

- **Pre/post hooks** на операции: создать объект, применить эффект, экспортировать
- **Macros**: записанные последовательности действий
- **Pipeline mode**: batch-обработка множества файлов
- **CLI**: управление из командной строки

### Примеры

- Автоматическая оптимизация при экспорте
- Batch-переименование слоёв по правилу
- Авто-расстановка элементов по сетке
- Генерация вариантов из параметров

---

## 5. Плагин runtime

### Что это

Плагины — **первоклассные граждане** с доступом к структурированному API.

- Не sandboxed до бесполезности
- Доступ к документ-модели, рендерингу, событиям
- Плагины на TypeScript (для UI) и Rust (для производительности)
- Менеджер плагинов из коробки

### Отличие от конкурентов

| Продукт | Плагин-модель | Доступ к ядру | Язык |
|---|---|---|---|
| Illustrator | CEP/UXP | Ограниченный | JS |
| Inkscape | Python extensions | Через Inkscape API | Python |
| Figma | Plugin API | Restricted | JS |
| **Rashamon** | Plugin runtime | Structured API | TS + Rust |

---

## 6. Linux-native workflow

### Что это

Rashamon думает как Linux-инструмент:

- **CLI**: `rashamon draw --export file.svg --format png`
- **Config files**: настройки в `~/.config/rashamon/`
- **Pipes**: `cat design.rdraw | rashamon draw --render`
- **System themes**: следует за темой системы
- **Font management**: использует системные шрифты

---

## 7. Shared Asset Graph

### Что это

Все продукты Rashamon используют **единый граф ассетов**:

- Символы, компоненты, стили — переиспользуемые между продуктами
- Изменение в Draw обновляет ссылку в Motion
- Централизованный asset manifest
