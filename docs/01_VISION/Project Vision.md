# Project Vision

> Создать профессиональную экосистему инструментов для творчества, которая принадлежит сообществу, думает системно и работает быстро.

---

## Purpose

Rashamon — это **не один продукт**, а экосистема специализированных инструментов для креативных профессионалов:

- **Rashamon Draw** — векторная графика, UI/UX дизайн, лэйауты
- **Rashamon Photo** — растровая обработка, фото, композитинг
- **Rashamon Motion** — анимация, моушн-дизайн

Каждый продукт решает свой класс задач, но все они работают на **общей платформе** с единым форматом проектов, системой ассетов и плагинами.

---

## Problem Statement

Существующие профессиональные инструменты имеют системные проблемы:

1. **Vendor lock-in** — проприетарные форматы, подписочная модель, невозможность миграции
2. **Платформенное неравенство** — Linux-пользователи системно игнорируются
3. **Монолитная архитектура** — невозможность кастомизации, тяжёлые обновления
4. **Отсутствие нефункциональности** — большинство инструментов не поддерживают branching history, семантические слои, автоматизацию
5. **Закрытая экосистема** — плагины существуют в песочнице вендора, не имеют доступа к ядру

---

## Vision Statement

Rashamon задуман как:

- **Экосистема**, а не клон одного продукта
- **Профессиональный инструмент**, а не хобби-проект
- **Открытая платформа**, а не закрытый монолит
- **Linux-first**, а не port-afterthought
- **Семантическая среда**, а просто холст с пикселями

---

## Key Decisions

| Решение | Обоснование |
|---|---|
| Экосистема продуктов | Каждый инструмент решает свой класс задач; общая платформа снижает дублирование |
| Open source по умолчанию | Контрибьюторы, доверие, форкабельность — это фича, а не угроза |
| Linux-first | Неосвоенная ниша; лояльное комьюнити; серьёзный gap на рынке |
| Семантический канвас | Структурированные данные вместо плоских пикселей/векторов — основа для автоматизации |
| Графовая история | Ветвящаяся история — уникальная возможность, которой нет в конкурентах |

---

## Assumptions

- Существует достаточное число профессионалов на Linux, готовых контрибьютить и использовать
- Tauri 2 обеспечит достаточную производительность для MVP
- Сообщество поддержит экосистемный подход вместо «ещё одного редактора»
